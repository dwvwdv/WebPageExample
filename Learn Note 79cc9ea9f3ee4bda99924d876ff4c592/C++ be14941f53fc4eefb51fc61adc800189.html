先來聊聊左值右值

左值(lvalue)

  • 可修改的左值
  • 不可修改的左值

右值(rvalue)

  • pure rvalue-純右值(字面量)
  • eXpiring Value-將亡值

++i是左值，i++是右值。

i++是i被拷貝出來(一個ti，而 ti = t+1)，最終回傳的是ti，ti本身沒有空間存放他，所以是右值，++i在本身上進行+1(i = i+1)，故是左值。

一個式子如果成立(能夠合法更動)，等號左邊的就是左值。

在C中，左值被定義為可修改的值，而在C++左值分為可修改的左值、不可修改的左值。

//不可修改的左值
const int num = 10; // num 為左值
num = 15;           // 但無法賦值

非A即B，不是左值的，就是右值，右值本身在記憶體區塊沒有空間存他，所以只存在於暫存器。

#include <stdio.h>

int main(void) {
    int n = 10;
    int *p = &n ;

    printf("n 的位址：%p\n", &n);
    printf("p 的位址：%p\n", &p);
    printf("*p的位址：%p\n", &(*p));
    return 0;
}

/*
	 n 的位址：0061FF1C
	 p 的位址：0061FF18
	*p 的位址：0061FF1C
*/

const修飾詞

const分為兩種

  • 常數(又或者稱呼為編譯期常數)
  • 只讀變數(又或者稱呼為執行期常數)

只有接受字面量初始化的值，才能進符號表，符號表的值是鎖死的，所以成為常數。

有一個例外，被volatile修飾的const哪怕接受字面量初始化，也不會進入符號表，是只讀變數。

編譯期間能確定初始值的，就是常數(除了添加volatile修飾狀況)。

const與指標

  • const int : int型態的一個常數。
  • const int * 、 int const * : 指向一個常數int型態的指標，指標本身可變動。
  • int * const : 指向int型態的常數指標，指標本身不可變動。
  • const int * const : 指向一個常數int型態的const指標，指標本身不可變動。

第二條，兩種是一樣的，都可以理解為指向const int 的一個指標，第三種則是屬於指向int的const指標，第四種是指向const int的const指標。

RAII機制

C++管理資源、避免洩漏的慣用法。利用的是C++建構物件最終會被銷毀的原則。RAII使用一個物件，建構時獲取對應資源，在物件生命週期內控制對資源的訪問始終保持有效，最後物件解構，釋放建構時取地的資源。

資源定義

資源是有限的，且對系統正常運作有一定作用，例如：Socket(套接字)、mutex(互斥鎖)、File Handle(文件句柄)、記憶體等等，但資源有限，所以每次使用都必須遵循一個步驟：

  • 申請資源
  • 使用資源
  • 釋放資源

前兩步缺一不可，最後一步如果不釋放，會導致資源短缺，從而洩漏。

右值引用

讓一個常引用指向一個將亡值，那麼將亡值的生命週期將會延長到與這個常引用相同長度。

#include <iostream>

using namespace std;

class Apple{
public:
	Apple(){
		int *App = new int[5];
		cout << "create App malloc space." << endl;
	}
	~Apple(){
		delete [] App;
		cout << "clear App malloc space." << endl;
	}
	int *App = NULL;
};
Apple getApple(){
	Apple a;
	return a;
}

void setApple(const Apple & a){
	cout << "const reference." << endl;
}

void setApple(Apple && a){
	cout << "right value reference." << endl;
}

int main(){
	const Apple & apple = getApple();
	setApple(apple);
	cout << "End1" <<endl;
	
	Apple a;
	setApple(a);
	cout << "End2" << endl;

	setApple(getApple());
	cout << "End3" << endl;
    
    return 0;
}
/*
create App malloc space.
const reference.
End1
create App malloc space.
const reference.
End2
create App malloc space.
right value reference.
clear App malloc space.
End3
clear App malloc space.
clear App malloc space.
*/
