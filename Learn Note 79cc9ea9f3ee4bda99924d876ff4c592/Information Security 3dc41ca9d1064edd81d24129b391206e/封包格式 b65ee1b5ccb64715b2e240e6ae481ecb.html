封包格式

Http/Https

Request

請求行

Requests method、URL、HTTP版本組成

example : GET /page.html HTTP/1.0

Requests Method

  • GET 簡易請求
  • POST 帶參數的請求
  • HEAD 伺服器不返回<body>內容的GET
  • PUT 帶參數的請求，但會把舊有請求蓋掉(用在更新
  • DELETE 刪除的這個頁面(但HTTP可在不通知客戶端下撤銷執行此方法
  • OPTIONS 查看伺服器端性能
  • TRACE 顯示伺服器端收到的requests封包，主要用於測試
  • CONNECT PUT請求的補充，進行局部更新

請求頭

  • HOST 地址
  • Accept MIME格式 ，Servlet用他判斷回傳合適的文件格式
  • User-Agent 客戶瀏覽器名稱
  • Host URL + port
  • Accept-Laungeuage 瀏覽器可以接受語言(en、en-us)
  • conection 用來告訴伺服器請求是否持續連接(HTTP 1.1默,認值為Keep-Alive，持續連接)
  • Cookie 儲存數據的地方，告知伺服器瀏覽器上儲存的數據
  • Referer 追蹤是從哪個網站來的
  • Content-Type request內容類型
  • Accept-Charset 瀏覽器可接受編碼，(google默認是ISO-8859-1)
  • Accept-Encoding 瀏覽器可接受編碼，是用來壓縮加速傳遞的
  • X-Forwarded-For 請求端ip

※請求頭中的同源政策

空行

請求頭後的空行，給帶參數的請求傳遞參數的

example: param = 100

Response

狀態行

HTTP版本、status code 組成

example: HTTP/1.1 200 OK

Status Code

1xx 請求收到，繼續處理

2xx 成功

3xx 重定向

4xx 客戶端錯誤

5xx 伺服端錯誤
