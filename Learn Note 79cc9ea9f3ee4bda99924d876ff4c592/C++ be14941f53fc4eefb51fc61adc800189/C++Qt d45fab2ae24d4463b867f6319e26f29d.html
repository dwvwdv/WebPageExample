connect、信號、槽理解

connect(& obj, action, & obj2, action2)

理解方式

因果關係觸發

因為a對c做了b事件，導致c觸發d事件

其中b扮演著信號，d扮演著槽函數，而a與c皆為物件

例如：我罵了你，導致你很難過

這個情境中a為我，c為你，b是罵人這件事，c因為被罵觸發了d很難過

同時

a做了b事件，同時c做了d事件

例如：點擊按鈕，關閉視窗。

這個情境中按鈕為a，點擊為b，視窗為c，關閉為d。

以上兩種理解方式我更傾向第一種，也更符合訊號與槽的感覺，不過看人習慣。

信號槽

Signal 與 Slot

可以是成員函數、全域函數、靜態函數、lambda表達式

槽函數要與信號對應(返回值、參數)

信號沒有返回值，槽函數就是void

例如：

void signal(int , QString );
void slot(int , QString );

槽函數參數是為了接收信號傳遞過來的數據

信號函數參數永遠不能比槽函數參數少

槽函數可以重載

跨窗口發訊號，使用emit

Connect

//Qt5 Signals Slots
connect(object, object::action, object2, object2::action2)

//多載寫法
void (subWidget::*signalReload_noparam)() = &subWidget::signalcloseWindow;
void (subWidget::*signalReload_2param)(int , QString) = &subWidget::signalcloseWindow;
connect(&subW,signalReload_noparam,this,&MainWindow::slotShowMe);
connect(&subW,signalReload_2param,this,&MainWindow::slotsubMsg);

//Qt4 Signals Slots
connect(object, SIGNAL(action()), object2, SLOT(action2));

//多載寫法
connect(object, SIGNAL(action()), object2, SLOT(action2));
connect(object, SIGNAL(action(int ,QString )), object2, SLOT(action2(int ,QString )));

Qt4的訊號槽看起來很簡潔，但有一個致命的缺陷導致後面Qt5在訊號槽這塊不使用宏定義(SIGNAL和SLOT)，因為一旦發生錯誤，Qt4是可能連報錯也不報錯的，難以debug。

對話框阻塞與非阻塞

阻塞是指還未關閉子窗口前，父窗口無法進行任何操作

非阻塞則是父子窗口都可進行操作

拿QDialog類舉例

阻塞寫法

connect(btn,&QPushButton::clicked,this,[=](){
	QDialog dlg(this);
	dlg.exec();
});

非阻塞寫法，Qt::WA_DeleteOnClose可以在關閉時馬上銷毀子窗口

connect(btn,&QPushButton::clicked,this,[=](){
	QDialog *dlg(this);
	dlg->setAttribute(Qt::WA_DeleteOnClose);
	dlg->show();
});
