Use Git

Git四區切換

# --system 整個系統
# --global 這個使用者
# --local 特定專案

git config --global user.name "dwvwdv"
git config --global user.email "dwvwdv@protonmail.com"
git config --global color.ui true

#可以查看是否設定成功
git config --list

git init #初始化

git add . #檔案進行修改後，紀錄變化狀態，寫到暫存區域
git commit -m "註釋" #提交版本，上傳到本地倉庫
#最後代碼分支合併 提交到遠程倉庫(github、gitlab等等) 線上master分支

git rm #git 的 rm
git rm --cached file #砍掉的是git本地倉庫裡的file

git mv #改名正確需使用 git 的 mv 不要使用 shell自帶的 mv，檔案會變為未跟蹤之前的狀態

git status #查看目前 git工作區 狀態
git log -2   #查看git最新的2個commit提交
git log --oneline #只簡易顯示每次提交提交註釋

git reset HEAD file #撤銷你對file進行的操作

git reset --hard [version-id] #直接指向版本ID號
git reset --hard HEAD^  #HEAD指向當前版本的上一個版本，HEAD^^則是上上版

git reflog #記錄了每次版本提交與回退，可以查到回到未退回的版本號HEAD，就能再回去


#當你寫的代碼量不足以完成一次提交但又有必須先放下工作時，存放到stash，下次調用
git stash save "註釋"
git stash list #查看保存列表與id
git stash pop stash_id #調用stash臨時空間裡面的文件，不加id則是恢復最新的stash
git stash clear
git stash drop stash_id

#遠程提交
git remote -v #查看遠端倉庫
git remote add origin git@github.com:dwvwdv/CodeToolBox.git #添加倉庫連結
git branch -M master #創建master分支
git push origin master #推上去master

#移除remote origin
git remote remove origin

git push -f #危險指令 強制更新 請確定倉庫裡所有的東西都在git控制裡，例如readme.md，此指令會完全覆蓋

#從遠端拉下來把現有檔案更新到最新進度
git pull
git merge origin/master

.git底下的index檔保存暫存區訊息

HEAD是一個指針，會指向目前要使用的版本，用來版本回退、前進。
