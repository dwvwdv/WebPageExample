C#運作

C#為微軟開發的語言，由C與C++衍伸而成，透過.NetFrameWork、.NET Core運行。

沒有多重繼承、記憶體管理、提供大量API調用。

和Java類似，由虛擬機編譯，但還是有很多不同的地方，Java是轉成JVM，而C#先編譯成中間語言(CIL)，再透過CLR(可以稱為.NET 虛擬機)運行，CLR這個運行庫會提供各種服務(記憶體管理、型態檢查等等)，由C#和.NET寫出來的程式碼就是IL。

中間語言(IL、CIL、MSIL)

在編譯.NET程式語言時，原始碼被翻譯成CIL碼，而不是基於特定平台或處理器的目標代碼。CIL是一種獨立於具體CPU和平台的指令集，它可以在任何支持.NET framework的環境下運行。CIL碼在運行時被檢查並提供比二進位代碼更好的安全性和可靠性。

即時編譯(JIT)

是指字節碼在CPU執行時被立即轉換成機器碼。轉換是在程序執行的過程中逐漸進行的。即時編譯技術提供了特定環境下的最佳化、運行時類型安全以及彙編驗證。為此，即時編譯器檢查程序集元數據中是否有非法訪問和句柄的衝突。

JIT出來的就是Native Code，可以執行。

MSIL(托管代碼)與非托管代碼

  • 運作在CLR上
  • 非托管代碼會直被編譯成機器碼
  • 托管代碼獨立於平台跟語言，更好實現不同語言平台間的兼容
  • 非托管代碼依賴平台與語言

過程

原始碼→編譯成CIL

打開程式的時候

CLR將CIL載入→CLR通過JIT把CIL轉換成Native Code執行

跨平台

不動code的前提下，去兼容其他平台的運行

Mono&Unity

C#早期只能運行在Windows，所以找上了Mono。

Mono為一個虛擬機，解釋執行CIL，透過JIT去轉換成組合語言，最後在目標平台運行，實現跨平台。

後來iOS認為這不安全，，JIT轉換需要底層支持動態代碼生成，意味著記憶體需要分配出可寫可讀取的區域，當一個程式能分配到一塊可寫可讀取的記憶體，容易造成允許任意代碼動態生成產生攻擊，所以被iOS禁止了。

IL2CPP&Unity

出現在JIT被iOS禁止後。

IL2CPP把C#的IL轉換成C++程式碼，然後通過各個平台的C++編譯器編譯成能執行的組合語言，再來就是熟悉的C++編譯過程了。

但仍然保留了IL2CPP VM ，這是為了GC管理跟線程創建等問題。
