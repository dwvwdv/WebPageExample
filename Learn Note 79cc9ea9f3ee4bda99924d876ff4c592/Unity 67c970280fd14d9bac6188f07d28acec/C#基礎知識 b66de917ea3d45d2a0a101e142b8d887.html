工具箱

裝箱操作-box

值類型隱式轉換Object類型或此值類型實現的任何街口類型的過程。

int a = 1;
object o = a;

o接受賦值，會在動態記憶體區(堆)開闢空間，拷貝一份a的值、一個同步塊索引、一個類型對象指針

拆箱操作-unbox

承上代碼

int b = (int) o;

檢查o箱中的值是否與b一致，一致的話賦值給b，不一致則報錯

裝箱範例

fun(object o){}//define fun(object )
fun(15);//call fun(object)
fun("str");//call fun(object)

範例這種狀況，更好的處理方式是function重載、泛型

ref 參數 與 out 參數差別

fun1(int );
fun2(ref int );
fun3(out int );

int a1 = 1;
fun1(a1);

int a2 = 2;
fun2(ref a2);

int a3 = 3;
fun3(out a3);

a2如果沒賦值，報錯，a3沒賦值，可運行

a2進fun2後沒賦值，可運行，a3進fun3後沒賦值，報錯

ref著重在傳入值，out著重在傳出值

StringBuilder

使用一塊較大的記憶體空間修改字串，用來改善字串拼接消耗性能問題

以下code每次拼接都會指向新的一塊空間，不斷產生垃圾

string str;
for(int i=0;i<10;i++)
	str += i;

使用StringBuilder，只需申請一次空間，參數不給有預設

StringBuilder builder = new StringBuilder(10);
for(int i=0;i<10;i++)
	builder += i;

StringBuilder(10)，超過會申請一塊更大的空間進行修改

標誌枚舉

  • 跳著定義，不使用預設01234逐增枚舉，使用 | 位運算去作多種選擇
  • 確保每個狀態疊加後的值都是不衝突的
  • 定義枚舉標誌[Flags]，不加也可以跑，但加了符合規範
  • 判斷時 用 &位運算得知哪幾個疊加了
[Flags]
enum status{
		die = 1,
		alive = 2,
		walk = 4,
		rest = 8,
		sleep = 16
}
/// 說明
/// status.alive | status.walk = 6

//  Console.WriteLine((status.alive | status.walk) & status.alive)
//  output : alive

GC回收

GC工作時，CPU會將大多數資源給GC，GC整理整個記憶體空間，乾淨的放到一塊，無用的放到一塊，這時候遊戲容易卡頓等等，整個應用都會受到影響，我們可以建議GC甚麼時候清理，但要不要做清理還得看GC本身。
