pacman套件管理

使用注意事項

在wsl中想要使用 pacman，要先初始化 Initialize keyring

但我在過程中遇到了glibc版本跟不上的問題，絕大多數指令都如下

version `GLIBC_2.34' not found

最後是以下指令解決

pacman -S glibc lib32-glibc

pacman.conf

裡面紀錄了由哪個Server提供套件，在此作修改

sudo vim /etc/pacman.conf

切換靜像過後，記得強制同步，否則導致數據丟失

pacman -Syy

定時清理安裝包緩存

刪除沒有被同步或沒有安裝的包

pacman -Sc

刪除所有緩存中的包，最為激進的方式

pacman -Scc

安裝oh-my-zsh

Shell

zsh安裝、oh-my-zsh安裝

pacman -Syu zsh
sh -c "$(curl -fsSL https://raw.githubusercontent.com/ohmyzsh/ohmyzsh/master/tools/install.sh)"

插件

終端提示

git clone git://github.com/zsh-users/zsh-autosuggestions ~/.zsh/plugins/zsh-autosuggestionsgit clone https://github.com/zsh-users/zsh-autosuggestions ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-autosuggestions

高亮

git clone https://github.com/zsh-users/zsh-syntax-highlighting.git ${ZSH_CUSTOM:-~/.oh-my-zsh/custom}/plugins/zsh-syntax-highlighting

autojump

git clone https://gitee.com/null_454_5218/autojump.git $ZSH_CUSTOM/plugins/autojump

pacman 安裝 autojump可能會遇到的問題

target not found: autojump
